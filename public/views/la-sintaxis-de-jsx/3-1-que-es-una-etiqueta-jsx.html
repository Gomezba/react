¿Qué es una etiqueta JSX?

Introducción

En esta sección trataremos de entender que es lo que estamos creando cuando estamos tratando una etiqueta en JSX.

Ya sabemos que JSX se termina transpilando a JavaScript y se transforma en React.createElement o su equivalente JSX runtime.

Pero ¿Qué es lo que devuelven estas funciones? Cuando trabajamos con HTML tenemos una sintaxis predefinida que podemos utilizar para crear nuestros documentos, gracias a esta sintaxis podemos utilizar etiquetas para crear un contenedor de elementos, podemos utilizar la etiqueta script para insertar un script, podemos utilizar las etiquetas meta, link o la etiqueta title, etc.

En esencia lo que nos proporciona HTML son una serie de etiquetas que representan componentes nativos y pre creados que podemos utilizar para componer nuestras páginas.

Diferencia entre etiquetas HTML y JSX

En JSX disponemos de las mismas etiquetas que en HTML con el mismo nombre. En ambos lenguajes existe una etiqueta de apertura <> y una etiqueta de cierre </>, o podemos crear etiquetas autocerrables como <br /> o <img />

En JSX es obligatorio el cierre

La única diferencia entre ambas sintáxis, es que mientras en ciertas etiquetas HTML podemos omitir su etiqueta de cierre y funcionaria, en JSX es obligatorio este cierre para absolutamente todas las etiquetas.

Por eso es importante que sepamos las pequeñas diferencias entre ambas, mientras que en HTML estamos utilizando una sintaxis que va ser interpretada por el navegador, en JSX estamos utilizando una sintaxis que se va transformar a JavaScript, y quitar la etiqueta de cierre será equivalente a quitar la llave de cierre de un objeto, es decir no podriamos cerrar el objeto correctamente.

¿En qué se transforma una etiqueta JSX?

Para ver realmente en que se transforma un etiqueta JSX lo primero que debemos de hacer es logearla por consola.

Con esto podemos ver, que es lo que devuelve de forma primigenia, es decir, primeramente lo que se almacena como objeto.

Transformación de una etiqueta JSX en Desarrollo

Arrancamos el servidor de desarrollo con el comando:

npm run dev

Podemos ver que lo que imprime realmente, es un objeto que representará un nodo del Virtual DOM.

Transformación de una etiqueta JSX en Producción

Creamos nuestra versión de producción:

npm run build

Levantamos nuestro servidor de producción:

npm run preview

Esto es un nodo del Virtual DOM directamente, es decir, que el Virtual DOM que estamos creando en React, realmente así es como se va reflejar esta etiqueta.

Obviamente aquí nos falta cierta información, es decir, cuando trabajamos con un Virtual DOM tenemos que tener identificado donde está este elemento renderizado, es decir, en que parte del Virtual DOM y en que parte del DOM real.

Por supuesto esto es un objeto que tiene una jerarquía y tiene que tener una forma directa de trasladarlo al DOM, hay que tener una seríe de indicativos que le permitan saber donde está renderizado ese elemento en el momento actual.

root.render()

Todo esto es lo que añade la función root.render() lo que hace es crear el Virtual DOM y posteriormente trasladarlo al DOM, es decir, lo primero que hace es crear el elemento, en nuestro caso un <img"> que está guardado en una variable llamada "img" que va representar al Virtual DOM , y una vez creado lo tiene que llevar al DOM original, pero tenemos que conectar ambos de alguna forma.

Con root.render(), se indica qué componente React debe ser renderizado dentro del contenedor raíz. Normalmente, este componente es el componente principal de tu aplicación, por ejemplo, <App />.

root.render() lo que hace es crear o actualizar el Virtual DOM a partir del componente que le pasamos. El Virtual DOM es una representación en memoria del DOM real. React se encarga de comparar (diffing) el nuevo Virtual DOM con la versión anterior para detectar los cambios, y luego actualiza eficientemente el DOM real, aplicando solo las modificaciones necesarias.

Nosotros simplemente declaramos que queremos renderizar un componente en un contenedor, y React, de manera declarativa, se encarga de gestionar todo el proceso de actualización del Virtual DOM y la reconciliación con el DOM real. Los detalles de cómo se maneja el Virtual DOM pueden variar entre versiones de React, pero ese proceso está abstraído para que no tengamos que preocuparnos por él.`
